# Kubernetes - Présentation

## Qu'est-ce que Kubernetes ?

Kubernetes (souvent abrégé **K8s**) est une plateforme d'orchestration de conteneurs. Son rôle est de déployer, exécuter et gérer des applications conteneurisées sur un ou plusieurs serveurs.

Plutôt que de démarrer manuellement des conteneurs sur chaque machine, Kubernetes permet de décrire **l'état souhaité** de l'application. Il se charge ensuite de maintenir cet état automatiquement.

Par exemple, il est possible de demander :

- exécuter 3 instances d'une application ;
- redémarrer automatiquement un conteneur en cas de panne ;
- répartir les instances sur plusieurs serveurs ;
- exposer l'application sur le réseau ;
- mettre à jour l'application sans interruption de service.

L'utilisateur décrit le résultat attendu, Kubernetes s'occupe de l'atteindre et de le maintenir.

---

## Pourquoi utiliser Kubernetes ?

Sans orchestrateur, la gestion d'applications conteneurisées devient rapidement complexe lorsque le nombre de services ou de serveurs augmente.

Kubernetes automatise notamment :

- le déploiement des applications ;
- le redémarrage des conteneurs défaillants ;
- la montée et la descente en charge (scaling) ;
- la répartition des charges entre les serveurs ;
- les mises à jour progressives (Rolling Update) ;
- les retours arrière (Rollback).

---

## Quelques notions importantes

### Cluster

Un **cluster** est un ensemble de machines sur lesquelles Kubernetes exécute les applications.

Il est généralement composé :

- d'un **Control Plane**, qui pilote le cluster ;
- d'un ou plusieurs **Worker Nodes**, qui exécutent les applications.

---

### Déclaratif

Kubernetes fonctionne selon un modèle **déclaratif**.

Au lieu d'exécuter des commandes du type :

> Démarre ce conteneur.

on décrit l'état souhaité :

> Je veux que cette application possède 3 instances.

Kubernetes compare en permanence l'état réel à l'état désiré et effectue les actions nécessaires pour corriger les écarts.

---

### Les ressources Kubernetes

Tout dans Kubernetes est représenté par une **ressource** (Resource).

Quelques exemples :

| Ressource | Rôle |
|-----------|------|
| Pod | Exécute un ou plusieurs conteneurs |
| Deployment | Gère le cycle de vie des Pods |
| Service | Permet d'accéder aux Pods |
| ConfigMap | Fournit de la configuration |
| Secret | Stocke des données sensibles |
| PersistentVolumeClaim | Demande un espace de stockage |

Ces ressources seront détaillées dans les pages suivantes.

## Comment Kubernetes fonctionne (vue simplifiée)

Le fonctionnement général peut être résumé ainsi :

1. Un utilisateur applique une configuration (`kubectl apply`).
2. Le Control Plane enregistre cette configuration.
3. Kubernetes détermine les actions nécessaires.
4. Les Pods sont créés sur les Worker Nodes.
5. Kubernetes surveille en permanence que l'état réel correspond à l'état demandé.

L'objectif est que le cluster soit **auto-réparateur** ("self-healing") : si un Pod disparaît, Kubernetes en recrée automatiquement un nouveau.

---

## Les principaux composants

Les composants principaux du cluster sont :

- **Control Plane**
  - API Server
  - Scheduler
  - Controller Manager
  - etcd

- **Worker Node**
  - kubelet
  - Container Runtime
  - kube-proxy
  - Pods

Une page spécifique décrit le rôle de chacun de ces composants.

---

## Ce qu'il faut retenir

- Kubernetes est un orchestrateur de conteneurs.
- Il fonctionne selon un modèle déclaratif.
- L'utilisateur décrit l'état souhaité, Kubernetes le maintient automatiquement.
- Les applications sont exécutées dans des Pods.
- Les autres objets (Deployment, Service, ConfigMap, etc.) permettent de gérer le cycle de vie des applications.
- Un cluster est composé d'un Control Plane et de Worker Nodes.
